我们可以创建一个方法实现所有操作
public void doPstm(String sql,Object[] params)
sql为String变量，存储SQL语句，params为Object类型数组，存储SQL语句中为占位符”?”赋值的数据。
举栗：SQL = "select * from user where name = ?"
“?”由PreparedStatement对象使用setXXX()方法进行赋值，
pstm.setString(1,"username")
整数1表示第一个占位符”?”，我们可以在同一SQL语句中多次使用占位符”?”，第二个参数为赋给占位符的值；
这样，SQL语句就相当于”select * from user where name = ‘username’”,这样做的好处使得查询变得灵活。

关于游标

游标是一种数据缓冲区，用来存放SQL语句执行的结果。
数据基础：游标是在先从数据表中检索出数据之后才能继续灵活操作的技术，类似于指针。

这两种对象的实例化都可以通过参数设定游标(concur)的行为动作，当然也可以忽略这两个参数。
参数type取值有三种：
ResultSet.TYPE_FORWORD_ONLY：结果集的游标只能向下滚动
ResultSet.YTPE_SCROLL_INSENSITIVE：结果集的游标可以上下移动，当数据库变化时结果集不变
ResultSet.TYPE_SCROLL_SENSITIVE：结果集的游标可以上下移动，当数据库变化时结果集同步改变
参数concurrency取值有两种：
ResultSet.CONCUR_READ_ONLY：不能用结果集更新数据库中的表
ResultSet.CONCUR_UPDATABLE：能用结果集更新数据库中的表
一些游标使用的方法：
next()：向下移动游标
previous()：向上移动游标
beforeFirst()：将游标移动到结果集初始位置，即第一行前，对应的有afterLast()方法
first()：将游标移动到结果集最后一行，对应的last()方法
getRow()：获取游标所指行号
absolute(int row)：将游标移到row指定行
还有一些判断方法，例如：isBeforeFirst(),不一一列举
